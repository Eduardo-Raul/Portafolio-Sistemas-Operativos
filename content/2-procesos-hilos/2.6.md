# Sistema de llamada con `_exit()` 


Como se ha visto en los anteriores temas, un proceso debe acabar de alguna manera, ya sea normal o anormal, loque se desea es que termine de forma normal y para esto, el proceso debe hacer una llamada al sistema _exit()



---

**PROTOTIPO DE LA FUNCIÓN PARA exit()**


```c
#include <unistd.h>

    void _exit (int status);
```

---


El argumento `status` define o guarda el estado de terminacion del proceso, el cual:
si es == 0 -> El proceso termino correctamente
si es != 0 -> Ocurrio algun error


---


# Sistema de llamada con `exit()` 


Otra forma de terminar el programa es con la funcion `exit()` la cual realiza varias acciones antes de terminar el proceso, pues se encarga de:

• libera recursos
• cierra archivos abiertos
• vacía buffers de salida
• notifica al proceso padre
• deja el proceso listo para ser eliminado.

Después internamente llama a _exit() para terminar la ejecucion del proceso y devuelve el valor de la variable status al proceso padre cuando se ocupa `wait()`.



---



```c
#include <stdlib.h>

    void exit (int status);
```


---




## Codigo de ejemplo

En el siguiente ejemplo, vemos un uso muy simple de la funcion `exit()`, en donde primero se imprimen dos mensajes, despues exit termina el proceso y con esto cierra buffers, archivos, recursos, etc. por lo que el ultimo mensaje nunca se llega a mostrar

```c
#include <stdio.h>
#include <stdlib.h>

int main(void) {

    printf("Inicio del programa\n");

    printf("Finalizando con exit()\n");

    exit(0);

    printf("Este mensaje nunca se imprime\n");

    return 0;
}
```


---


## Salida en pantalla
Inicio del programa

Finalizando con exit()

## Codigo de ejemplo


En el siguiente ejemplo un uso muy interesante de `_exit()`, pues con esta funcion se termina inmediatamente el proceso, esto causara que el mensaje NO aparezca en pantalla, esto es debido a que la funcion `printf()` usa un buffer, pero `_exit()` no vacia ese buffer antes de terminar el proceso, por lo que no se muestra ningun mensaje 



```c
#include <stdio.h>
#include <unistd.h>

int main(void) {

    printf("Inicio del programa");

    _exit(0);
}
```

## Aprendizaje y mejoras


En este tema vimos mas a fondo como se termina un proceso, sabiendo que existen dos formas de terminarlos mediante las funciones `exit()` y `_exit()`, las cuales a pesar que tienen la misma funcion y el mismo final, ambos tienen un rpoceso muy distinto para terminar un proceso, pues uno se encarga de terminarlo instantaneamente, mientras que el otro hace pasos adicionales para tener un poco mas de orden en el sistema, pues cierra todo lo ocupado por el proceso y lo deja listo para ser eliminado por el sistema, lo cual es algo a tener en cuenta cuando se trabaja con multiples procesos que pueden acceder a multiples recursos diferentes
