# Sistema de llamada con `wait()` 


Cuando un proceso crea un nuevo hijo con `fork()`, tanto el padre como el hijo siguen su ejecucion de manera concurrente, como consecuencia uno de los procesos, puede ser el padre o el hijo, puede terminar primero que el otro, esto puede llegar a no ser un problema, pero para un mejor manejo, se tiene a la funcion `wait()`, con ella el padre puede esperar a que su proceso hijo o alguno de sus procesos en especifico, termine su ejecucion y le notifique al padre para que recupere su estado de terminacion.



Esto se consigue de la siguiente forma; al llamar a `wait()`, se suspende temporalmente la ejecucion del proceso, hasta que ocurre lo siguiente:  


* Uno de los procesos hijos termina su ejecucion.
* Un proceso hijo se detiene.
* El proceso que invocó a wait() recibe una señal.

Algo a considerar, es el valor de retorno pues: 

* El valor de retorno es positivo y corresponde al PID del proceso si no hay errores.
* En caso de error, se retorna -1 y se establece el valor del error en la variable `errno`.

algunos de esos valores relevantes pueden ser:

* ECHILD: el proceso no tiene hijos a los cuales esperar .
* EINTR: la llamada fue interrumpida por una señal recibida.


---


**PROTOTIPO DE LA FUNCIÓN PARA wait()**


```c
#include <sys/types.h>
#include <sys/wait.h>

    pid_t wait (int *stat_loc);
```


---


El parametro `stat_loc` es un apuntador a una variable de tipo entero, en donde el kernel almacena la informacion del estado de terminacion del proceso, este valor puede analizarse con las macros definidas en `<sys/wait.h>`, es decir, `stat_loc` es una variable en donde se guarda la informacion de como termino el proceso y al ser un numero se pueden ocupar las macros para analizar como acabo, por ejemplo:

* `WIFEXITED(*stat_loc)`: Evalua si el proceso termino de forma normal.

* `WEXITSTATUS(*stat_loc)`: Si el proceso termino de forma normal, obtiene el valor retornado por `exit()`.

* `WIFSIGNALED(*stat_loc)`: Evalua si el proceso termino debido a una señal recibida.

* `WTERMSIG(*stat_loc)`: Si el proceso termino debido a una señal, obtiene el valor de dicha señal.

# Sistema de llamada con `waitpid()` 


Cuando se requiere esperar a un proceso hijo en especifico, se necesita un control mas adecuado, por lo que en estos casos debemos ocupar `waitpid()`, esta llamada tiene el mismo comportamiento que `wait()`, pero con la diferencia que ahora se suspende la ejecucion del proceso padre hasta que el proceso hijo especificado finaliza o hasta que ocurre un evento controlado.


---


**PROTOTIPO DE LA FUNCIÓN PARA waitpid()**


```c
#include <sys/types.h>
#include <unistd.h>

     pid_t waitpid (pid_t pid, int *wstatus, int options);
```


---


El parametro `pid` puede tomar los siguientes valores:

* -1 Para esperar a cualquier hijo.
* `>0` Para esperar al hijo que tenga ese PID exacto.
* 0 Para esperar hijos del mismo grupo.
* `<0` Para esperar hijos de un grupo especifico.


El parametro `wstatus` cumple la misma funcion que `stat_loc` para guardar la infromacion de termino, al igual que se analiza con los mismos macros 


El parametro options permite modificar el comportamiento de la llamada mediante la combinacion de las siguientes opciones o banderas:

• 0: comportamiento normal sin opciones especiales.
• WEXITED: espera por hijos que hayan terminado.
• WSTOPPED: espera por hijos que hayan sido detenidos por una señal.
• WNOHANG: no bloquea la ejecucion del padre, retorna inmediatamente si ningún hijo ha terminado.
• WNOWAIT: no elimina al hijo de la tabla de procesos.
• WUNTRACED: permite detectar hijos detenidos temporalmente.
• WCONTINUED: permite detectar cuando un proceso detenido vuelve a continuar su ejecucion.

## Codigo de ejemplo

En el siguiente ejemplo, despues de llamar a la funcion `fork()`, El hijo imprime su PID y retorna un numero 5, mientras esto sucede, el padre se detiene en wait() hasta que termina el hijo, una vez esto, se continua la ejecucion del padre, el cual tambien imprime su PID y con **WEXITSTATUS** puede imprimir el valor de retorno del hijo, el cual se guarda en la variable **estado**

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>

int main(void) {

    pid_t hijo;
    int estado;

    hijo = fork();

    if (hijo == 0) {

        /* Código del proceso hijo */

        printf("Soy el hijo. PID=%ld\n", (long)getpid());

        exit(5);

    } else {

        /* Código del proceso padre */

        wait(&estado);

        printf("Soy el padre. PID=%ld\n", (long)getpid());

        printf("El hijo ya termino\n");

        printf("Codigo de salida del hijo: %d\n", WEXITSTATUS(estado));
    }

    return EXIT_SUCCESS;
}
```


---


## Salida en pantalla
Soy el hijo. PID=27162

Soy el padre. PID=27161
El hijo ya termino
Codigo de salida del hijo: 5

## Codigo de ejemplo


En el siguiente ejemplo se crean dos hijos, pero solo se espera a la terminacion de uno en especifico, cada uno tiene su PID unico, pero al usar waitpid le indicamos al padre cual debe esperar a que termine, no importa si el otro termina antes su ejecucion, en este caso ocupamos la funcion `sleep(2)` para darle un pequeño retraso al proceso 1 y se note más la espera de terminación



```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>

int main(void) {

    pid_t hijo1, hijo2;
    int estado;

    hijo1 = fork();

    if (hijo1 == 0) {

        printf("Soy el hijo 1. PID=%ld\n", (long)getpid());

        sleep(2);

        exit(1);
    }

    hijo2 = fork();

    if (hijo2 == 0) {

        printf("Soy el hijo 2. PID=%ld\n", (long)getpid());

        exit(2);
    }

    /* El padre espera solo al hijo 1 */

    waitpid(hijo1, &estado, 0);

    printf("El hijo 1 termino\n");

    printf("Codigo de salida: %d\n", WEXITSTATUS(estado));

    return EXIT_SUCCESS;
}
```

---



## Salida en pantalla

Soy el hijo 1. PID=28875
Soy el hijo 2. PID=28876
El hijo 1 termino
Codigo de salida: 1

## Aprendizaje y mejoras


En este tema vimos como podemos esperar a que un proceso ya sea cualquiera o uno en especifico termine su ejecucion para poder avanzar con la ejecucion principal, algo muy util para el control de ejecucion de los procesos en el sistema, pues es de gran ayuda el poder esperar a que un proceso en especifico termine su ejecucion para poder trabajar con sus datos, al igual que combinando todas las configuraciones de comportamiento, como tambien el saber como termino la ejecucion de un proceso, nos ayuda a mantener un control adecuado de los procesos

Una propuesta para mejorar en la practica de este tema seria implementar el uso de mas opciones de comportamiento, como el uso de WNOHANG para ver como se comportan los procesos cuando no se bloquea la ejecucion del padre, o el uso de distintos valores en la variable `pid` para ver como se comporta el sistema cuando espera al primer proceso en acabar
