# Creación de procesos con fork()  


Cuando se ejecuta un programa en una computadora, el sistema operativo crea un proceso.  


Los procesos pueden aparecer de dos maneras:  


* El sistema operativo los crea para hacer tareas internas.
* El usuario los crea cuando abre algun programa como un navegador o una aplicacion.  


Ademas que un programa puede crear otros procesos mientras se esta ejecutando.  


---


## Funcion `fork()`  


En sistemas GNU/Linux esto se hace usando la funcion `fork()`.  


Esta funcion hace que un proceso existente (proceso padre) cree otro nuevo proceso (proceso hijo).  


Entonces, cuando se ejecuta `fork()`:  

1. El sistema operativo crea un nuevo proceso hijo.
2. El hijo recibe una copia identica del codigo del padre.
3. Ambos procesos continuan ejecutandose en simultaneo.


---


## Valor de retorno

La diferencia es que cada uno recibe un valor de retorno distinto:

- El padre recibe el identificador (PID) del hijo.
- El hijo recibe 0.

Gracias a esto el programa puede saber cual es cual.

---

## Importancia

Este mecanismo de diferenciacion es esencial en la programacion concurrente, ya que permite al desarrollador controlar el flujo de ejecucion y definir las responsabilidades del padre y del hijo.

---

## Prototipo de la funcion

**PROTOTIPO DE LA FUNCIÓN**

```c
#include <sys/types.h>
#include <unistd.h>

pid_t fork(void);
```

---

## Código de ejemplo
En el siguiente codigo tanto el padre como el hijo ejecutan la asignacion x = 1


```c
    #include <sys/types.h>
    #include <stdlib.h>
    #include <unistd.h>
        int main(void) {
            int x = 0;
            fork();
            x = 1;
            return EXIT_SUCCESS;
        }
```
Tras la ejecucion de fork() ahora existen dos procesos independientes, el padre y el hijo, ambos continuan su ejecucion despues de la llamada, por lo que ambos ejecutan la asignacion de x = 1, aunque cada proceso cuenta con un espacio diferente, por lo que la variable x es distinta en cada uno

---

## Código de ejemplo
En el siguiente codigo se utiliza el valor de retorno para distinguir entre el padre y el hijo, cada uno modifica la variable de manera distinta
```c
    #include <stdio.h>
    #include <stdlib.h>
    #include <sys/types.h>
    #include <unistd.h>
        int main(void) {
            int x = 0;
            pid_t pid;
            pid = fork();
            if (pid == 0) {
            /* Código ejecutado por el proceso hijo */
            x = 5;
            printf("Hijo: PID=%ld, x=%d\n", (long)getpid(), x);
            } else {
            /* Código ejecutado por el proceso padre */
            x = 10;
            printf("Padre: PID=%ld, x=%d\n", (long)getpid(), x);
            }
            return EXIT_SUCCESS;
        }
```


## Salida en pantalla 


Padre: PID=1331, x=10


Hijo: PID=1332, x=5

## Aprendizaje y mejoras


En este tema vimos como se crean los procesos en el sistema, que a pesar de que el hijo es una copia del padre, gracias al valor de retorno de la funcion fork() se puede trabajar de manera independiente con ambos, esto es muy importante para la programacion concurrente pues se pueden dividir las tareas entre los procesos, la forma en que mejoraria esta practica seria implementando la creacion de mas procesos en los cuales se resuelvan operaciones mas complejas para mostrar como a pesar de tener la misma variable, cada proceso trabaja con ella de manera independiente sin afectar a los otros procesos
