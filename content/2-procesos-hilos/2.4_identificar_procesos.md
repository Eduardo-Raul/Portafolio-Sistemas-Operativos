# Identificacion de procesos con PID() y PPID()


Todos los procesos tienen un numero identificador asociado y unico denominado PID (Process Identifier), el cual es un numero entero positivo asignado por el kernel, de la misma forma, cada proceso guada la referencia del proceso que lo creó (del padre) y este se denomina PPID (Parent Process Identifier), para obtener esos valores debemos ocupar dos funciones 

---

**PROTOTIPO DE LA FUNCIÓN PARA PID**

```c
#include <sys/types.h>
#include <unistd.h>

    pid_t getpid(void);
```

---

**PROTOTIPO DE LA FUNCIÓN PARA PPID**

```c
#include <sys/types.h>
#include <unistd.h>

    pid_t getppid(void);
```

---


El tipo de dato `pid_t` representa al identificador de un proceso, el cual es un numero entero con signo   


La funcion `getpid()` devuelve el identificador del prodeso que la manda a llamar, miestras que la funcion `getppid()` retorna el identificador del padre del proceso que la esta mandando a llamar 


---


Ademas de esa relacion de padre-hijo, los procesos tambien pueden organizarse en grupos, los cuales permiten gestionar de forma conjunta procesos relacionados, por ejemplo los que esten asociados a una misma terminal, para identificar el grupo al que pertenece un proceso, se utiliza la llamada `getpgrp()`, esta funcion devielve el identificador del grupo de procesos


---


**PROTOTIPO DE LA FUNCIÓN PARA PGRP**

```c
#include <sys/types.h>
#include <unistd.h>

    pid_t getpgrp(void);
```

---



## Codigo de ejemplo

En el siguiente ejemplo, despues de llamar a la funcion `fork()`, el proceso padre y su proceso hijo imprimen sus respectivos identificadores 

```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h>

int main(void) {

    pid_t hijo;
    hijo = fork();

    if (hijo == 0) {
        /* Código ejecutado por el proceso hijo */
        fprintf(stdout, "Soy el hijo, PID=%ld\n", (long)getpid());
    } else if (hijo > 0) {
        /* Código ejecutado por el proceso padre */
        fprintf(stdout, "Soy el padre, PID=%ld\n", (long)getpid());
    } else {
        /* Error al crear el proceso */
        perror("fork");
        return EXIT_FAILURE;
    }
    return EXIT_SUCCESS;
}
```

## Salida en pantalla
Soy el padre, PID=635


Soy el hijo, PID=636




## Codigo de ejemplo


En el siguiente ejemplo se muestra una cadena de procesos, donde cada proceso crea un solo hijo y el padre deja de crear nuevos procesos, teniendo como resultado una estructura del tipo P0 -> P1 -> P2 -> P3 -> ... -> Pn. En cada iteracion el hijo es el que continua el ciclo y cada proceso tiene solo un hijo



```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h>

int main(void) {

    pid_t hijo;
    int n = 5;
    for (int i = 0; i < n; i++) {
        hijo = fork();
        if (hijo > 0) {
            /* El padre deja de crear más procesos */
            break;
        }
        fprintf(stderr,"Proceso PID=%ld, PPID=%ld\n",(long)getpid(), (long)getppid());
    }
    return EXIT_SUCCESS;
    }
```

## Salida en pantalla

Proceso PID=16447, PPID=16446

Proceso PID=16448, PPID=16447

Proceso PID=16450, PPID=16448

Proceso PID=16451, PPID=16450

Proceso PID=16452, PPID=16451


## Aprendizaje y mejoras


En este tema vimos que un proceso puede conocer su propio numero de identidad o mejor conocido como PID, al igual que el PID de su creador (padre), para esto se ocupan las funciones; `getpid()` para saber el identificador de un proceso y `getppid()` para saber el identificador de quien lo creó, esto resulta util para que podamos visualizar mejor la relacion de padre-hijo que mantienen los procesos, al igual que ayuda al sistema operativo a oorganizar y controlar a los procesos de forma adecuada.

Una propuesta para entender mejor este tema seria la practica de crear una cadena de procesos como el ejemplo anterior e incluir el uso de operaciones y variables para ver como cada proceso sigue una ejecucion independiente y sus operaciones no afectan a los resultados de otros procesos
