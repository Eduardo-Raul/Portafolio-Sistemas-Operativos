# Estado Zombie 


Un proceso se denomina Zombie cuando este ya termino su ejecución, pero su proceso padre no ha recogido su estado de salida mediante `wait()` o `waitpid()`, cuando se encuentra en este estado, no consume CPU ni memoria del programa, pero si ocupa una entrada en la tabla de procesos, es decir sigue apareciendo en el sistema, lo cual podria llegar a ser un problema, pues si se acumulan muchos procesos zombie, el sistema tendra problemas poque la tabla de procesos tiene un limite

¿Qué significa esto?

Cuando un proceso hijo termina su ejecución:

el sistema no lo elimina inmediatamente
primero se guarda informacion importante como:
    su PID 
    su codigo de salida
    datos estadisticos

toda esta informacion se guarda para que el proceso padre pueda consultarla con una llammada a `wait()` o `waitpid()`, mientras el padre no recoja esta información, el hijo se queda en este estado zombie, pues es como si ya estuviera muerto pero todavia queda el registro suyo dentro del sistema




¿Cómo aparece un zombie?

De manera simple, aparecen de la siguiente forma:
El padre crea un hijo con`fork()`
El hijo termina su proceso con `exit()`
El padre no usa `wait()`
El sistema deja al hijo en estado zombie esperando a que el padre recoja los resultados de terminación



---



## Codigo de ejemplo

En el siguiente ejemplo, vemos como se crea un proceso zombie, para esto:
Primero se crea un nuevo proceso hijo con `fork()`
despues el hijo imprime un mensaje y termina su ejecución con `exit(0)`
el padre sigue vivo por 30 segundos pero nunca llama a la funcion `wait()`
El kernel conserva su información y marca al hijo como proceso zombie durante esos 30 segundos 



```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <unistd.h> 

int main(void) {

    pid_t pid;
    pid = fork();

    if (pid == 0) { /* Proceso hijo */
        printf("Hijo terminado. PID=%ld\n", (long)getpid());
        exit(EXIT_SUCCESS);
    } else { /* Proceso padre */
        printf("Padre en ejecución. PID=%ld\n",(long)getpid());
        sleep(30); /* El padre NO llama a wait() */
    }
    return EXIT_SUCCESS;
} 
```

## Como evitar zombies


Principalmente se evita la creacion de procesos zombie utilizando las llamadas `wait()` o `waitpid()` en el proceso padre, para que se recoja la informacion de termino de un proceso hijo y pueda ser eliminado por el sistema.

Pero si no ocurriera esto, el sistema esta diseñado con un mecanismo para evitar que se creen zombies, pues si el proceso padre termina su ejecución, o la termina antes que los hijos, estos se vuelven "huerfanos" y el sistema les asigna automaticamente un nuevo proceso padre, con el PID 1 (init o systemd) este nuevo padre ejecuta `wait()` automaticamente para que no quede ningun zombie en el sistema



## Codigo de ejemplo


En el siguiente ejemplo se muestra la forma correcta para evitar procesos zombies, mediante el uso de `wait()` en el proceso padre para recoger los datos de el hijo uma vez que acabe su ejecución 



```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/types.h>
#include <sys/wait.h>
#include <unistd.h>
int main(void) {
    pid_t pid;
    int status;
    pid = fork();

    if (pid == 0) { /* Proceso hijo */
        printf("Hijo terminado. PID=%ld\n", (long)getpid());
        exit(EXIT_SUCCESS);
    } else { /* Proceso padre */
        wait(&status); /* Recolecta al hijo */
        printf("Padre: hijo recolectado\n");
    }
    return EXIT_SUCCESS;
}
```

## Aprendizaje y mejoras


En este tema vimos la importancia de recoger los datos guardados al momento de que un proceso hijo termina su ejecucion con el uso de `wait()` y `waitpid()`, pues a pesar de que no se siguen ejecutando ni ocupando CPU o memoria, siguen en el sistema y por consecuencia en la tabla de procesos, ocupando lugares importantes para poder crear nuevos procesos que si esten trabajando, al igual que vimos el proceso que sigue el sistema para protegerse de esto, asignandole al proceso 1 (init o systemd) todos esos hijos sin padre para que no se conviertan en zombies 

Una propuesta para mejorar lo aprendido en la practica seria la creación de mas procesos en estado zombie para poder ver como se comporta el sistema al crearse multiples procesos zombie
