# Tuberias con nombre - FIFO


Las tuberias con nombre, tambien llamadas `FIFO` (First In First Out), son mecanismos de comunicación entre procesos muy parecidos a las tuberías sin nombre, pero con una diferencia importante:

- las FIFO si tienen nombre dentro del sistema de archivos

Por esta razon tambien son conocidas como `Named Pipes`


---


# ¿Como funcionan las FIFO?


El funcionamiento de una FIFO es practicamente igual al de una tuberia normal, un proceso escribe información y otro proceso la recibe


`Proceso A ----> [FIFO] ----> Proceso B`


- El proceso A escribe datos
- La FIFO transporta los datos
- El proceso B los recibe o los muestra


Su nombre `First In First Out` significa:

- el primer dato que entra es el primero que sale

Es decir, los datos se reciben en el mismo orden en el que fueron enviados


---


# Diferencia entre PIPE y FIFO


Aunque ambas sirven para comunicar procesos, existen algunas diferencias importantes:


| PIPE | FIFO |
|---|---|
| No tiene nombre | Tiene nombre |
| Solo existe durante la ejecución | Existe como archivo especial |
| Comunicación entre padre e hijo | Comunicación entre procesos independientes |
| Se crea con `pipe()` | Se crea con `mkfifo()` |


Una de las mayores ventajas de las FIFO es que permiten comunicar procesos completamente independientes, es decir, los procesos no necesitan tener relacion padre-hijo ni compartir un ancestro en comun


---


# Creación de una FIFO


Las FIFO se crean usando la función `mkfifo()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/stat.h>

int mkfifo(const char *pathname, mode_t mode);
```


---


Los parametros de `mkfifo()` son:

- pathname: nombre o ruta de la FIFO
- mode: permisos que tendrá el archivo especial


Entonces si tenemos:

```c
mkfifo("canal", 0666);
```

- `"canal"` es el nombre de la FIFO
- `0666` representa los permisos de lectura y escritura para todos los usuarios


---


# Apertura de una FIFO


Una FIFO se puede abrir igual que un archivo usando la función `open()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <fcntl.h>
#include <sys/types.h>
#include <sys/stat.h>

int open(const char *pathname, int flags);
```


---


Los parametros de `open()` son:

- pathname: nombre de la FIFO
- flags: modo de apertura


Algunas opciones comunes para `flags` son:

- O_RDONLY: abrir solo para lectura
- O_WRONLY: abrir solo para escritura
- O_RDWR: abrir para lectura y escritura


---


# Codigo de ejemplo


En el siguiente ejemplo veremos una comunicación usando una FIFO, en donde un proceso escribe un mensaje y otro proceso independiente lo recibe e imprime en pantalla

Para lograr esto:
1. Se crea una FIFO llamada `canal`
2. El programa escritor abre la FIFO en modo escritura
3. El programa lector abre la FIFO en modo lectura
4. El mensaje se manda usando `write()`
5. El lector recibe el mensaje usando `read()`


---


## Programa escritor

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/stat.h>

int main()
{
    int fd;

    char mensaje[] = "Hola desde FIFO";

    mkfifo("canal", 0666);

    fd = open("canal", O_WRONLY);

    write(fd, mensaje, sizeof(mensaje));

    close(fd);

    return EXIT_SUCCESS;
}
```


---


## Programa lector

```c
#include <stdio.h>
#include <stdlib.h>
#include <fcntl.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/stat.h>

int main()
{
    int fd;

    char buffer[100];

    fd = open("canal", O_RDONLY);

    read(fd, buffer, sizeof(buffer));

    printf("Mensaje recibido: %s\n", buffer);

    close(fd);

    return EXIT_SUCCESS;
}
```


---


## Salida en pantalla


Mensaje recibido: Hola desde FIFO



---


A diferencia de las tuberías sin nombre, aqui no fue necesario usar `fork()` porque ambos programas son independientes


---


## Aprendizaje y mejoras


En este tema vimos el funcionamiento de las tuberías con nombre FIFO y como permiten la comunicación entre procesos independientes mediante un archivo especial dentro del sistema, tambien observamos la diferencia respecto a las tuberías sin nombre y como es posible enviar y recibir datos usando las funciones `open()`, `read()` y `write()`


Una propuesta para mejorar la practica seria implementar una comunicación bidireccional utilizando multiples FIFO para que ambos procesos puedan enviar y recibir información al mismo tiempo, al igual que combinar el uso de FIFO con hilos para poder atender varias comunicaciones concurrentes de manera simultanea para ver como funciona una ejecucion mas compleja de la programación concurrente
