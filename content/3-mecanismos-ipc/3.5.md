# Información de IPC por medio de comandos del sistema


Los sistemas UNIX y GNU/Linux proporcionan diferentes comandos que permiten visualizar y administrar los mecanismos IPC creados dentro del sistema operativo

Estos comandos ayudan a monitorear recursos como:

- memoria compartida
- semaforos
- colas de mensajes


Esto es muy importante para identificar recursos activos, verificar permisos o eliminar recursos IPC que quedaron ocupados despues de ejecutar programas


---


# Comando ipcs


El comando `ipcs` permite mostrar información sobre los mecanismos IPC activos en el sistema


Su sintaxis general es:


```bash
ipcs
```


---


Este comando puede mostrar:

- colas de mensajes
- segmentos de memoria compartida
- conjuntos de semaforos


La salida normalmente incluye:

- identificadores IPC
- propietarios
- permisos
- tamaño de memoria
- numero de procesos conectados


---


## Ejemplo de uso de ipcs


```bash
ipcs -m
```


Esto muestra solamente los segmentos de memoria compartida activos


```bash
ipcs -q
```


Muestra las colas de mensajes activas


```bash
ipcs -s
```


Muestra los semaforos activos en el sistema


---


# Comando ipcrm


El comando `ipcrm` permite eliminar recursos IPC manualmente


Esto es util cuando un programa termina incorrectamente y deja recursos IPC ocupados dentro del sistema


La sintaxis general es:


```bash
ipcrm [opcion] id
```


---


Algunas opciones importantes son:

- -m para eliminar memoria compartida
- -q para eliminar colas de mensajes
- -s para eliminar semaforos


Ejemplo:


```bash
ipcrm -m 12345
```


Esto elimina el segmento de memoria compartida con identificador 12345


---


# Comando ipcmk


El comando `ipcmk` permite crear recursos IPC directamente desde la terminal


Por ejemplo:


```bash
ipcmk -M 1024
```


Esto crea un segmento de memoria compartida de 1024 bytes


Otros ejemplos:


```bash
ipcmk -Q
```


Crea una cola de mensajes


```bash
ipcmk -S 1
```


Crea un conjunto de semaforos


---


# Comando lsipc


El comando `lsipc` permite mostrar información detallada sobre recursos IPC en sistemas GNU/Linux modernos


Su funcionamiento es similar a `ipcs`, pero con una salida mas organizada y detallada


Ejemplo:


```bash
lsipc
```


Este comando muestra:

- recursos IPC activos
- limites del sistema
- tamaños maximos
- identificadores y permisos


---


## Aprendizaje y mejoras


En este tema vimos como los comandos del sistema permiten visualizar y administrar mecanismos IPC desde la terminal de GNU/Linux, tambien se aprendio el uso de herramientas como `ipcs`, `ipcrm`, `ipcmk` y `lsipc` para monitorear recursos IPC y eliminar recursos que ya no sean necesarios

Una mejora interesante seria combinar estos comandos con programas escritos en C para observar en tiempo real como cambian los recursos IPC mientras los procesos se ejecutan
