# Comunicación mediante tuberias


La comunicacion entre procesos es fundamental para poder intercambiar datos, para poder comunicar procesos, debemos considerar si estos se van a comunicar en la misma maquina o si estan en diferentes maquinas


Las tuberías son mecanismos de comunicación entre dos o mas procesos en la misma maquina en sistemas UNIX System V

Su funcionamiento es muy parecido al de una tubería real, pues un proceso escribe información por un extremo y otro proceso la recibe del otro lado, es decir, un proceso genera información y otro proceso la muestra


`Proceso A ----> [Tubería] ----> Proceso B`

- El proceso A escribe datos
- La tubería transporta los datos
- El proceso B los lee o los muestra



# Tuberias sin nombre 


Las tuberías sin nombre, tambien llamadas `pipes` son mecanismos de comunicación proporcionados por todos los sistemas UNIX, se llaman anonimas porque:

- no tienen nombre dentro del sistema de archivos
- solo existen mientras los procesos están ejecutandose

Su facilidad de uso les da una ventaja para implementarlas de manera sencilla, pero presentan dos limitaciones importantes:

1. Los datos solo fluyen en una dirección.
2. Pueden usarse solo entre procesos que tengan un ancestro en común

Es decir, normalmente la tuberia e creada por un proceso que invoca a la funcion `fork()` y es usada entre Padre --> hijo, con una comunicación unidireccional, es decir en un solo sentido van a viajar los datos, si se quisiera una comunicacion en ambos sentidos se tendrian que ocupar dos tuberias 




Los hilos son una forma de ejecutar varias tareas al mismo tiempo dentro de un mismo programa, al igual que una alternativa mas ligera, pues a diferencia de los procesos, los hilos comparten la misma memoria y los mismos recursos del programa al que pertenecen

Una forma muy intuitiva de comprenderlo es imaginar que un proceso es como una oficina completa trabajando, mientras que un hilo sería una persona trabajando dentro de esa misma oficina


El uso de hilos resulta adecuado cuando se requiere
Ejecutar multiples tareas concurrentes
Hacer programas mas rapidos
Compartir datos de forma mas eficiente
Aprovechar procesadores con varios nucleos

y todo esto dentro del mismo proceso

## Diferencias entre hilos y procesos


| Característica | Proceso | Hilo |
| :--- | :--- | :--- |
| Memoria | Independiente | Compartida |
| Tamaño | Más pesado | Más ligero |
| Creación | con `fork()` | con `pthread_create()` |
| Variables | No comparte variables | Comparte variables |
| Rapidez | Mas lento de crear | Mas rápido de crear |


Algo importante de recalcar es con la cuestion de la memoria, pues para los procesos, cada uno tiene su propia memoria, por lo que si un proceso cambia una variable `x = 10` los demas procesos no ven este cambio

Sin embargo, los hilos si comparten memoria entre ellos, por lo que si un hilo cambia una variable `x = 10` puede afectar a todos los hilos del mismo programa, es por esto que los hilos son más rapios, pero tambien más delicados de manejar

# 2.8.2 Creación de hilos


Para la creacion de hilos en GNU/Linux, se ocupa una libreria especial llamada `<pthread.h>` 

---
PROTOTIPO DE LA FUNCIÓN
```c
#include <pthread.h>

int pthread_create (pthread_t *thread, const pthread_attr_t
*attr, void* (*start_routine) (void *), void *arg);
```
---

## ¿Qué significa cada parámetro?

| Parámetro| Función |
| :--- | :--- |
| `thread` | Guarda el ID del hilo creado |
| `attr` | Define atributos del hilo |
| `start_routine` | Funcion que ejecutará el hilo |
| `arg` | Argumento enviado a la función |


---

## Como terminar un hilo

Para que un hilo termine su ejecucion, debe suceder alguno de estos casos:

- Se llama a la función `pthread_exit()` especificando un valor de estado final, el cual esta disponible para los otros hilos del mismo proceso llamando a la funcion `pthread_join()`
- Realizar un `return` al final de la funcion en `start_routine`, lo cual equivale a llamar a la función `pthread_exit()`
- Es cancelado invocando a la función `pthread_cancel()`
- El hilo principal ejecuta un `return` desde main causando la terminación de todos los hilos


---


## Errores en los hilos

Al crear un nuevo hilo, la funcion `pthread_create()` retorna un 0 en caso de exito o un numero de error en caso contrario y el contenido de `*thread` queda indefinido, los errores que pueden retornarse en `errno` son:

- EAGAIN: insuficientes recursos para crear un hilo o el limite de hilos en el sistema es alcanzado
- EINVAL: el atributo es inválido
- EPERM: Politica de planificacion no permitida y los parametros de attr

## Codigo de ejemplo

En el siguiente ejemplo veremos el funcionamiento de los hilos, para esto, se creara un conjunto de hilos y cada uno calculará el factorial de un numero que se envia como argumento desde la linea de comandos, los datos s eguararan en una estructura y una vez que terminen de calcular el factorial de cada numero, se imprimen los resultados

Algo importante de mencionar es que para compilar programas que ocupen la libreria de `<pthread.h>` se debe ejecutar el comando `gcc -Wall chilos.c -lpthread -o chilos` y para ejecutar el programa para pasar los argumentos se hace de la forma `./chilos <numero1> <numero2> ... <numero10>` en este caso se ejecutara el programa con los argumentos 2, 5 y 10



```c
#include <pthread.h>
#include <stdio.h>
#include <stdlib.h>

typedef struct dhilos {
    int id;
    long prod;
} DHILOS;

DHILOS pm_hilos[10];

void *factorial (void *valor);

int main (int argc, char *argv[])
{
 pthread_t tid[argc-1];
 pthread_attr_t attr;
 int i;
 if (argc<2)
 {
  perror ("Uso: ./chilos <entero1> <entero2> ... \n");
  exit(1);
 }
 // Crear hilos
   for (i=0;i<argc-1;i++)
   {
        pthread_attr_init(&attr); // Atributos predeterminados
     pm_hilos[i].id=i+1;
     pm_hilos[i].prod=atol(argv[i+1]);
     pthread_create(&tid[i], &attr, factorial, &pm_hilos[i]);
    }
   // espera el proceso principal a cada hilo creado
   for (i=0;i<argc-1;i++)
    pthread_join(tid[i], NULL); // Esperar finalización, semejante a wait-cuando usa fork
  for (i=0;i<argc-1;i++)
    printf("Hilo-%d calculó Factorial de: %s = %ld\n", pm_hilos[i].id,argv[i+1],pm_hilos[i].prod);
 return EXIT_SUCCESS;
 }

void *factorial (void *valor)
{
 int i=1, prod=1;
 DHILOS *datos;
 datos=(DHILOS*)valor;
 while (i <= datos->prod)
  prod *= (i++);
 datos->prod=prod;
 pthread_exit(&(datos->prod));
 }
```

Hilo-2 calculó Factorial de: 5 = 120
Hilo-3 calculó Factorial de: 10 = 3628800

## Aprendizaje y mejoras





En este tema vimos el funcionamiento de los hilos en los procesos, comprendiendo la diferencia entre estos y como podemos ocuparlos como una alternativa más rapida y ligera, aunque con un cuidado más demandante, pues sus diferencias importan mucho y hacen que debamos trabajar de diferente manera, por ejemplo con el uso de la memoria, pues los procesos tienen una memoria distinta de cada uno, por lo que hacer cambios en una variable no afecta a los demas procesos, mientras que los hilos ocupan memoria compartida, por lo que un cambio en una variable afecta directamente a todos los hilos, una caracterisitca muy poderosa y tambien muy peligrosa que debe usarse con cuidado, ya que gracias a estas caracteristicas, los hilos trabajan más rapido, pensados para realizar multiples tareas en simultaneo dentro de un mismo proceso 

Una propuesta para mejorar lo aprendido en la practica seria la implementacion de procesos e hilos para ver mas a detalle como trabajan en conjunto al ocupar mas de un proceso y a su vez varios hilos para cada proceso, haciendo multiples tareas en simultaneo y explorando mas a fondo el como se utilizan los nuceos de un procesador para ejecutar tareas más complejas como esta.
