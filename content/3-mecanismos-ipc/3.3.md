# Memoria compartida


La memoria compartida es uno de los mecanismos de comunicación entre procesos mas rapidos en sistemas UNIX y GNU/Linux

Su funcionamiento consiste en permitir que dos o mas procesos accedan a la misma región de memoria al mismo tiempo


A diferencia de otros mecanismos IPC como tuberias o colas de mensajes, la memoria compartida evita copias adicionales de información, ya que todos los procesos trabajan directamente sobre el mismo espacio de memoria


Esto permite:

- mayor velocidad de comunicación
- compartir grandes cantidades de datos
- reducir el uso de operaciones de entrada/salida


---


# Funcionamiento de la memoria compartida


El sistema operativo crea un segmento de memoria especial que puede ser enlazado por diferentes procesos

Cada proceso puede:

- escribir información
- leer datos compartidos
- modificar contenido comun


Aunque algo importante es que la memoria compartida no incluye sincronización automaticamente, por lo que normalmente se combina con semaforos para evitar condiciones de carrera


---


# Creacion de memoria compartida


La creación de un segmento de memoria compartida se realiza mediante la función `shmget()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmget(key_t key, size_t size, int shmflg);
```


---


Los parametros de `shmget()` son:

- key: llave unica de la memoria compartida
- size: tamaño del segmento
- shmflg: permisos y opciones de creación


Por ejemplo:

```c
shmget(clave, 1024, IPC_CREAT | 0666);
```

indica:

- crear un segmento de 1024 bytes
- usar permisos de lectura y escritura
- crear la memoria si no existe


El valor de retorno es:

- un identificador de memoria compartida si tiene éxito
- -1 si ocurre un error


---


# Enlace de la memoria compartida


Para poder utilizar el segmento creado, el proceso debe enlazarlo a su espacio de direcciones usando `shmat()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/ipc.h>
#include <sys/shm.h>

void *shmat(int shmid, const void *shmaddr, int shmflg);
```


---


Esta función devuelve un apuntador a la región de memoria compartida


Ejemplo:

```c
char *memoria;

memoria = (char *) shmat(shmid, NULL, 0);
```


Con esto el proceso ya puede leer y escribir directamente sobre la memoria compartida


---


# Liberacion de memoria compartida


Cuando el proceso termina de utilizar la memoria compartida debe separarla con `shmdt()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/ipc.h>
#include <sys/shm.h>

int shmdt(const void *shmaddr);
```


---


Para eliminar completamente el segmento de memoria compartida se utiliza `shmctl()` con la opción `IPC_RMID`


Ejemplo:

```c
shmctl(shmid, IPC_RMID, NULL);
```


---


## Codigo de ejemplo


En el siguiente ejemplo veremos como un proceso hijo escribe un mensaje en memoria compartida y el proceso padre lo lee posteriormente


---


```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/shm.h>
#include <sys/types.h>
#include <sys/wait.h>

int main()
{
    int shmid;
    key_t clave;
    char *memoria;

    clave = ftok(".", 'M');

    shmid = shmget(clave, 1024, IPC_CREAT | 0666);

    memoria = (char *) shmat(shmid, NULL, 0);

    if (fork() == 0)
    {
        strcpy(memoria, "hola desde memoria compartida");
    }
    else
    {
        wait(NULL);

        printf("Mensaje recibido: %s\n", memoria);

        shmdt(memoria);

        shmctl(shmid, IPC_RMID, NULL);
    }

    return EXIT_SUCCESS;
}
```


---


## Salida en pantalla


Mensaje recibido: hola desde memoria compartida

## Aprendizaje y mejoras


En este tema vimos como funciona la memoria compartida como mecanismo IPC y como diferentes procesos pueden acceder al mismo espacio de memoria para compartir información rapidamente

Tambien se aprendio el uso de las funciones `shmget()`, `shmat()`, `shmdt()` y `shmctl()` para crear, utilizar y eliminar segmentos de memoria compartida

Una mejora interesante para profundizar en este tema seria combinar memoria compartida con semaforos para sincronizar correctamente el acceso a los datos y evitar problemas de concurrencia entre multiples procesos
