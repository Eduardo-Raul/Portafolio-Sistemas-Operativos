# Cola de mensajes


Las colas de mensajes son mecanismos IPC que permiten la comunicación entre procesos mediante el envio y recepción de mensajes organizados en una cola

A diferencia de las tuberias, las colas de mensajes permiten que los procesos se comuniquen sin necesidad de estar relacionados directamente, ya que el kernel administra los mensajes enviados


En los sistemas UNIX derivados de System V, las colas de mensajes son utilizadas para:

- enviar datos entre procesos
- sincronizar tareas
- comunicar procesos independientes
- almacenar mensajes temporalmente


---


# Funcionamiento de las colas de mensajes


Una cola de mensajes funciona de manera similar a una fila donde los procesos colocan mensajes y otros procesos los reciben posteriormente

El sistema operativo almacena los mensajes dentro del kernel hasta que otro proceso los lea


Cada mensaje normalmente contiene:

- un tipo de mensaje
- datos o contenido


Esto permite que los procesos puedan leer mensajes especificos dependiendo de su tipo


---


# Creacion de una cola de mensajes


La creación de una cola de mensajes se realiza mediante la función `msgget()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/msg.h>

int msgget(key_t key, int msgflg);
```


---


Los parametros de `msgget()` son:

- key: llave unica de la cola
- msgflg: permisos y opciones de creación


Por ejemplo:

```c
msgget(clave, IPC_CREAT | 0666);
```

indica:

- crear la cola si no existe
- usar permisos de lectura y escritura


El valor de retorno es:

- un identificador de cola si tiene éxito
- -1 si ocurre un error


---


# Estructura de un mensaje


Para utilizar colas de mensajes se debe definir una estructura especial


```c
struct mensaje {
    long tipo;
    char texto[100];
};
```


Donde:

- tipo identifica el tipo del mensaje
- texto almacena la información enviada


El campo `tipo` es obligatorio y siempre debe ser de tipo `long`


---


# Envio de mensajes


Para enviar mensajes a la cola se utiliza la función `msgsnd()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/msg.h>

int msgsnd(int msqid, const void *msgp, size_t msgsz, int msgflg);
```


---


Los parametros de `msgsnd()` son:

- msqid: identificador de la cola
- msgp: apuntador al mensaje
- msgsz: tamaño del mensaje
- msgflg: opciones adicionales


Ejemplo:

```c
msgsnd(msqid, &msg, sizeof(msg.texto), 0);
```


Esto envía el contenido almacenado en `msg.texto` hacia la cola de mensajes


---


# Recepción de mensajes


Para leer mensajes desde la cola se utiliza la función `msgrcv()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/msg.h>

ssize_t msgrcv(int msqid, void *msgp, size_t msgsz, long msgtyp, int msgflg);
```


---


Los parametros mas importantes son:

- msqid: identificador de la cola
- msgp: apuntador donde se guardará el mensaje
- msgsz: tamaño del mensaje
- msgtyp: tipo de mensaje a recibir


Por ejemplo:

```c
msgrcv(msqid, &msg, sizeof(msg.texto), 1, 0);
```

indica:

- recibir mensajes de tipo 1
- guardar la información en `msg`


---


# Eliminacion de una cola de mensajes


Para eliminar completamente una cola de mensajes se utiliza `msgctl()` con la opción `IPC_RMID`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/msg.h>

int msgctl(int msqid, int cmd, struct msqid_ds *buf);
```


---


Ejemplo:

```c
msgctl(msqid, IPC_RMID, NULL);
```


Esto elimina completamente la cola de mensajes del sistema


---


## Codigo de ejemplo


En el siguiente ejemplo veremos como un proceso hijo envía un mensaje usando una cola de mensajes y el proceso padre lo recibe posteriormente


---


```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/msg.h>
#include <sys/types.h>
#include <sys/wait.h>

struct mensaje {
    long tipo;
    char texto[100];
};

int main()
{
    int msqid;
    key_t clave;
    struct mensaje msg;

    clave = ftok(".", 'Q');

    msqid = msgget(clave, IPC_CREAT | 0666);

    if (fork() == 0)
    {
        msg.tipo = 1;

        strcpy(msg.texto, "hola desde cola de mensajes");

        msgsnd(msqid, &msg, sizeof(msg.texto), 0);
    }
    else
    {
        wait(NULL);

        msgrcv(msqid, &msg, sizeof(msg.texto), 1, 0);

        printf("Mensaje recibido: %s\n", msg.texto);

        msgctl(msqid, IPC_RMID, NULL);
    }

    return EXIT_SUCCESS;
}
```


---


## Salida en pantalla


Mensaje recibido: hola desde cola de mensajes


---


## Aprendizaje y mejoras


En este tema vimos como funcionan las colas de mensajes como mecanismos IPC para permitir la comunicación entre procesos mediante mensajes organizados por el kernel, tambien se aprendio el uso de las funciones `msgget()`, `msgsnd()`, `msgrcv()` y `msgctl()` para crear, enviar, recibir y eliminar colas de mensajes

Una mejora interesante seria implementar multiples tipos de mensajes para que varios procesos puedan comunicarse simultaneamente usando diferentes prioridades o categorias de información
