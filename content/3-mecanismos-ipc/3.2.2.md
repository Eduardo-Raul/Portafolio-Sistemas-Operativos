# Semaforos en derivados de System V


Los semaforos son mecanismos de sincronización usados para controlar el acceso a recursos compartidos entre procesos, evitando problemas como condiciones de carrera o accesos simultaneos incorrectos

En los sistemas UNIX derivados de System V, los semaforos forman parte de los mecanismos IPC y permiten coordinar procesos independientes que se ejecutan al mismo tiempo


Un semaforo funciona como una variable entera controlada por el kernel, donde los procesos pueden:

- incrementar su valor
- disminuir su valor
- esperar hasta que un recurso este disponible


Los semaforos son utilizados principalmente para:

- sincronizar procesos
- controlar regiones criticas
- implementar exclusion mutua
- coordinar acceso a memoria compartida


---


# Funcionamiento basico de un semaforo


Un semaforo puede imaginarse como un contador protegido por el sistema operativo

Por ejemplo:

- si el valor del semaforo es mayor que 0, un proceso puede continuar
- si el valor es 0, el proceso queda bloqueado hasta que otro proceso libere el recurso


Las operaciones mas comunes sobre un semaforo son:

- wait o P: disminuye el semaforo
- signal o V: incrementa el semaforo


El kernel garantiza que estas operaciones sean atomicas, evitando que dos procesos modifiquen el semaforo al mismo tiempo


---


# Creacion de semaforos en System V


Para trabajar con semaforos en System V se utilizan varias funciones especiales definidas en la biblioteca `<sys/sem.h>`


La creación de un conjunto de semaforos se realiza mediante la función `semget()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/sem.h>

int semget(key_t key, int nsems, int semflg);
```


---


Los parametros de `semget()` son:

- key: llave unica del semaforo
- nsems: numero de semaforos del conjunto
- semflg: permisos y opciones de creación


Por ejemplo:

```c
semget(clave, 1, IPC_CREAT | 0666);
```

indica:

- crear un conjunto de 1 semaforo
- usar permisos de lectura y escritura
- crear el semaforo si no existe


El valor de retorno de `semget()` es:

- un identificador del conjunto de semaforos si tiene éxito
- -1 si ocurre un error


---


# Operaciones sobre semaforos


Las operaciones sobre los semaforos se realizan con la función `semop()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/sem.h>

int semop(int semid, struct sembuf *sops, size_t nsops);
```


---


El parametro `sops` define la operación que se realizará sobre el semaforo mediante la estructura `sembuf`


```c
struct sembuf {
    unsigned short sem_num;
    short sem_op;
    short sem_flg;
};
```


Donde:

- sem_num: numero del semaforo dentro del conjunto
- sem_op: operación a realizar
- sem_flg: opciones adicionales


Los valores mas comunes de `sem_op` son:

- -1 para disminuir el semaforo
- 1 para incrementar el semaforo
- 0 para esperar hasta que el valor sea 0


---


# Inicializacion de un semaforo


Para asignar un valor inicial a un semaforo se utiliza la función `semctl()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>
#include <sys/sem.h>

int semctl(int semid, int semnum, int cmd, ...);
```


---


Una de las operaciones mas usadas es `SETVAL`, que permite inicializar el semaforo


Ejemplo:

```c
semctl(semid, 0, SETVAL, 1);
```

Esto asigna el valor 1 al semaforo

## Codigo de ejemplo


En el siguiente ejemplo veremos como un proceso hijo entra a una región critica utilizando un semaforo System V


---


```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/ipc.h>
#include <sys/sem.h>
#include <sys/types.h>
#include <sys/wait.h>

int main()
{
    int semid;
    key_t clave;
    struct sembuf p = {0, -1, 0};
    struct sembuf v = {0, 1, 0};

    clave = ftok(".", 'S');

    semid = semget(clave, 1, IPC_CREAT | 0666);

    semctl(semid, 0, SETVAL, 1);

    if (fork() == 0)
    {
        semop(semid, &p, 1);

        printf("Hijo entrando a region critica\n");
        sleep(3);
        printf("Hijo saliendo de region critica\n");

        semop(semid, &v, 1);
    }
    else
    {
        wait(NULL);
        semctl(semid, 0, IPC_RMID);
    }

    return EXIT_SUCCESS;
}
```


---


## Salida en pantalla


```text
Hijo entrando a region critica
Hijo saliendo de region critica
```

## Aprendizaje y mejoras


En este tema vimos como los semaforos permiten sincronizar procesos y proteger recursos compartidos usando mecanismos IPC derivados de System V

Tambien se aprendio la creación de semaforos, la inicialización de valores y las operaciones de bloqueo y liberación mediante `semop()`

Una mejora para profundizar en el tema seria implementar varios procesos compitiendo por la misma región critica para observar como el semaforo evita accesos simultaneos incorrectos, al igual que combinar los semaforos con memoria compartida para construir aplicaciones concurrentes mas completas
