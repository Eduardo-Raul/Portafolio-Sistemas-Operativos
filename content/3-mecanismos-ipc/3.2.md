# Mecanismos IPC derivados de System V


Los mecanismos IPC (Inter Process Communication) derivados de System V son herramientas que permiten la comunicación y sincronización entre procesos dentro de una misma maquina en sistemas UNIX


Estos mecanismos fueron desarrollados para permitir que distintos procesos puedan:

- compartir información
- sincronizar tareas
- intercambiar datos
- coordinar la ejecución de procesos


Los mecanismos IPC mas utilizados de System V son:

- Memoria compartida
- Semaforos
- Colas de mensajes


Todos estos mecanismos utilizan identificadores especiales llamados llaves o `keys` para poder acceder a los recursos compartidos


---


# ¿Por que son importantes los IPC?


Cuando varios procesos trabajan al mismo tiempo, muchas veces necesitan compartir información o coordinar tareas entre ellos


Por ejemplo:

- un proceso puede generar datos
- otro proceso puede procesarlos
- y otro proceso puede mostrarlos


Los mecanismos IPC permiten que esta comunicación se haga de manera organizada y segura


---


# Caracteristicas de los IPC de System V


Los mecanismos IPC derivados de System V tienen algunas caracteristicas importantes:

- permiten comunicación entre procesos independientes
- los recursos pueden permanecer aun despues de que un proceso termine
- utilizan identificadores internos del sistema
- requieren llaves o `keys` para acceder a los recursos


Otra ventaja importante es que diferentes procesos pueden acceder al mismo recurso IPC aunque no tengan relación padre-hijo


---


# Recursos IPC


Cuando se crea un mecanismo IPC, el sistema operativo genera un identificador interno para administrarlo


Por ejemplo:

- una memoria compartida
- una cola de mensajes
- un semaforo


Cada uno de estos recursos tiene:

- una llave (`key`)
- un identificador interno (`id`)
- permisos de acceso


---


# 3.2.1 Llaves


Las llaves o `keys` son valores utilizados por el sistema para identificar y permitir el acceso a los recursos IPC


Las llaves funcionan como nombres o identificadores compartidos entre procesos, permitiendo que distintos programas puedan acceder al mismo recurso IPC


---


# ¿Como funcionan las llaves?


La idea es sencilla:

- un proceso crea un recurso IPC usando una llave
- otro proceso usa la misma llave para acceder a ese recurso


Por ejemplo:


Proceso A ----> crea memoria compartida con una key

Proceso B ----> usa la misma key para acceder



Sin las llaves seria muy dificil que procesos independientes pudieran encontrar el mismo recurso dentro del sistema


---


# Tipo de dato key_t


Las llaves normalmente utilizan el tipo de dato:

```c
key_t
```


Este tipo es utilizado por las funciones IPC para representar una llave unica dentro del sistema


---


# Generación de llaves con ftok()


La función mas utilizada para generar llaves es `ftok()`


---


PROTOTIPO DE LA FUNCIÓN
```c
#include <sys/types.h>
#include <sys/ipc.h>

key_t ftok(const char *pathname, int proj_id);
```


---


Los parametros de `ftok()` son:

- pathname: ruta de un archivo existente
- proj_id: identificador del proyecto


La función combina ambos valores para generar una llave unica (`key_t`)


---


# Ejemplo sencillo de ftok()


En este ejemplo:

1. Se utiliza el archivo `"archivo.txt"`
2. Se utiliza el identificador `65`
3. `ftok()` genera una llave unica
4. La llave se guarda en la variable `llave`


Esa llave puede utilizarse despues para acceder a mecanismos IPC como memoria compartida, semaforos o colas de mensajes

```c
#include <stdio.h>
#include <sys/types.h>
#include <sys/ipc.h>

int main()
{
    key_t llave;

    llave = ftok("archivo.txt", 65);

    printf("Llave generada: %d\n", llave);

    return 0;
}
```


---


# Consideraciones importantes


Para que `ftok()` funcione correctamente:

- el archivo indicado en `pathname` debe existir
- distintos procesos deben usar el mismo archivo y el mismo `proj_id`
- la llave generada no siempre es completamente unica en todo el sistema


Por esta razon el sistema operativo tambien utiliza identificadores internos para administrar cada recurso IPC


---


# Relación entre key e id


Es importante entender que:

- la `key` sirve para localizar el recurso
- el `id` es el identificador interno que asigna el sistema operativo


Es decir, de manera mas simple:

key ----> busca el recurso

id ----> administra el recurso internamente


---


## Aprendizaje y mejoras


En este tema vimos los mecanismos IPC derivados de System V y como permiten la comunicación y sincronización entre procesos independientes mediante recursos compartidos del sistema operativo, tambien observamos el uso de las llaves (`keys`) y la función `ftok()` para generar identificadores que permiten acceder a recursos IPC como memorias compartidas, semaforos y colas de mensajes


Una propuesta para mejorar la practica seria implementar ejemplos completos utilizando memoria compartida y semaforos para demostrar como varios procesos pueden compartir datos y sincronizar tareas concurrentes utilizando una misma llave IPC
